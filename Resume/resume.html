Server/Client TCP connection with Boost(C++) and FastAPI(Python) for NinjaStrike Game

Started: March 2025 - Present

Goal: To integrate C++ and Python by building a networked game architecture using Boost for the C++ client and server, FastAPI for comunication between database and C++ server.

  • Currently, the game supports basic player movement; shooting mechanics are yet to be implemented.
  • FastAPI passes the username and previous position from the database to the server.
  • The C++ client captures key inputs (e.g., "W", "A") and sends corresponding messages (e.g., "WA") asynchronously to the server using a dedicated networking thread, while a separate thread handles real-time rendering.
  • On the server side, the main thread asynchronously listens for incoming messages, while another thread performs game state calculations.
  • The server periodically (every 10 seconds) updates the player's position in the database via FastAPI.
  • FastAPI also integrates with Flask Web and Tkinter-based application to handle user registration and login functionality.
